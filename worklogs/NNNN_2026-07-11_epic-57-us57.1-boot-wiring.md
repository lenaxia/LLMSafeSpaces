## Key Decisions

- **Fail-closed over fail-open**: KMS construction failure returns nil, boot guard refuses to start. Previous draft's "log + fallback to local" was rejected by review as a security downgrade on the root of trust.
- **AWS SDK lazy-loads credentials**: bad credentials file is a runtime failure (first Encrypt/Decrypt), not a boot-time failure. Documented; chart's Helm `required` guards ensure the K8s Secret object exists.
- **Master-secret mount retained under KMS**: still consumed by `NewRedisDEKCache` for the volatile DEK cache (D5). KMS protects durable Postgres tables only.
- **//nolint:gocyclo on Load**: the .golangci.yml comment references it as grandfathered but it was never applied. Added it to match the documented baseline.
