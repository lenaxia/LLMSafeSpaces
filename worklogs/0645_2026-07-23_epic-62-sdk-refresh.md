## Context

An external project integrating against the Python SDK flagged SDK drift as their root-cause blocker. Their design rested on assumptions that were either wrong on the specifics or correct-but-unprovable. This worklog covers the implementation of the epic definition, OpenAPI spec refresh, Python SDK parity (sync + async), TypeScript SDK parity, session queue surface, and the EA3 contract test.

## Reviewer-found bugs (fixed in this PR)

1. **`list_bindings` returned wrong type** — API returns `{"workspaceIds": [...], "bindings": [...]}`, not bare `list[str]`. Fixed to extract `workspaceIds`. (Verified: `user_provider_credentials.go:417-420`)

2. **`create()` crashed on 207 MultiStatus** — API returns `{"credential": {...}, "bindWarning": "..."}` when auto-bind partially fails. Fixed to unwrap `credential` field. (Verified: `user_provider_credentials.go:183-188`)

3. **`create_auto_apply` test used invalid `targetType`** — `"workspace"` is rejected by the handler; valid values are `"all"`, `"user"`, `"org"`. Fixed test. (Verified: `admin_provider_credentials.go:445-448`)

4. **Dead `UserSettings` dataclass** — `_UserSettingsAPI.get()` returns `dict`, never `UserSettings`. Removed.
