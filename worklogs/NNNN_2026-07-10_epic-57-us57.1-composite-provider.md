## Key Decisions

- **Three-way `unwrapPrefix` router**: own-prefix → decode, foreign-prefix → `ErrNotMyCiphertext`, no-prefix → legacy passthrough. The legacy passthrough is the load-bearing backward-compat guarantee for every existing deployment.
- **`ErrNotMyCiphertext` distinct from `ErrDecryptionFailed`**: routing signal vs. genuine failure. Without this distinction, multi-provider dispatch would log spurious failures on every prefix mismatch.
- **`knownForeignPrefixes` is explicit, not dynamic**: package-level slice per README-LLM §3 ("Explicit Over Implicit"). Adding a provider means editing this list — visible at the call site, no init-order magic.
- **Composite stops on first non-`ErrNotMyCiphertext`**: a corrupt own-prefix row must NOT continue dispatch (would generate N-1 spurious calls). Pinned by `TestCompositeProvider_DecryptStopsOnPrefixMatch_NotErrNotMyCiphertext`.
- **Test fixes route through `provider.Decrypt` not raw `DecryptSecret`**: tests the real interface, removes coupling to the prefix format. Future prefix format changes won't break these tests.
- **British→American spelling**: CI `misspell` lints require American English (`behavior`, `recognized`). Applied repo-wide across the new files.
- **`var _ = errors.Is` hack removed**: dead code from an earlier test draft; flagged by reviewer, removed before merge.
