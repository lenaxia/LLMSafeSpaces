## Work Completed

### Go 1.25.12 bump

- `go.mod` `go` directive bumped from `1.25.11` to `1.25.12`
- 11 CI workflow `go-version` pins updated
- Fixes GO-2026-5856 (Invoking Encrypted Client Hello privacy leak in crypto/tls)
- Dockerfiles use unpinned `golang:1.25-bookworm` — no change needed

### AWSKMSProvider implementation (`pkg/secrets/kms_aws_provider.go`)

- `AWSKMSProvider{client, keyID}` struct implementing `RootKeyProvider`
- `Encrypt` → KMS Encrypt API call, wraps output with `aws-kms:v1:` prefix
- `Decrypt` → strips prefix, KMS Decrypt API call
- `unwrapKMSCiphertext` — prefix check returns `ErrNotMyCiphertext` for foreign prefixes; corrupt base64 returns `ErrDecryptionFailed`; no legacy fallback (KMS ciphertexts always carry the prefix from day one)
- Uses `bytes.HasPrefix` (not the test-only `hasPrefix` helper that caused the first CI failure)

### Tests (`pkg/secrets/kms_aws_provider_test.go`)

8 tests against a fake KMS server (`httptest.Server` implementing enough of the Encrypt/Decrypt JSON API to exercise the SDK client):
- `RoundTrip_EncryptThenDecrypt`
- `Encrypt_WrapsWithAwsKmsV1Prefix`
- `PrefixMismatch_ReturnsErrNotMyCiphertext`
- `Decrypt_KMSUnavailable_ReturnsError`
- `Decrypt_Throttled429_ReturnsError`
- `AuthFromCredentialsFile`
- `Decrypt_CorruptBase64_ReturnsErrDecryptionFailed` (added per review feedback)
- `Encrypt_KMSUnavailable_ReturnsError` (added per review feedback)
