# US-52.9: Inference-Relay Worker Tests

**Epic:** 52 — Comprehensive Test Coverage
**Layer:** Unit (vitest) + integration
**Effort:** Medium (2d)
**Depends on:** —
**Status:** ⛔ Superseded

> **Superseded by Epic 60 (2026-07-12).** The Cloudflare Worker at `workers/inference-relay/` was removed because Zen now blocks CF Worker egress IPs. There is no Worker code to test. See `design/stories/epic-60-remove-cf-worker-relay/README.md`.

---

## Problem

`workers/inference-relay/` was the Cloudflare Worker that proxied
inference traffic from opencode-relay to the upstream LLM provider. It had
exactly one test file (`src/index.test.ts`) covering the secret-validation
path and the happy-path proxy. The worker is the **LLM cost surface** — a
bug here either costs real money (orphaned requests, missing rate-limit,
double-billing) or breaks every paying user's inference silently.

Untested code paths in the worker:

| Path | File | Risk |
|---|---|---|
| Rate limiting (per-key, per-IP) | `src/index.ts` | No rate limit = cost runaway |
| Usage logging (tokens → billing) | same | No log = free inference |
| Circuit breaker (upstream-down detection) | same | No breaker = 502 storm on upstream failure |
| Relay-mode fallback (free → paid tier switch) | same | No fallback = users stuck on broken free tier |
| Streaming responses (SSE passthrough) | same | Dropped stream = silent truncation |
| Secret rotation (old + new valid simultaneously) | same | Rotation breaks active requests |
| Request body size limit | same | No limit = worker OOM |

## Scope

**In:**
- Vitest unit tests for each of the 7 paths above, using the existing
  `handleRequest(req, env)` pattern from `src/index.test.ts:6`.
- One integration test (`workers/inference-relay/src/integration.test.ts`)
  using Miniflare to run the worker against a stubbed upstream and verify
  end-to-end behaviour including streaming.

**Out:**
- Load testing (Epic 33).
- Real LLM provider calls — tests stub the upstream.
- Refactoring the worker for testability beyond extracting pure functions
  where strictly necessary.

## Intent-Declared Test Inventory

### `src/ratelimit.test.ts` (new)

| Test | What | Value added | Failure mode protected | Expected |
|---|---|---|---|---|
| `allows up to limit per key` | N requests same key | Per-key limit | Limit ignored = cost runaway | N succeed |
| `rejects over limit per key` | N+1th request | Same | Same | 429 with Retry-After |
| `limit resets after window` | Time advance | Window reset | Permanent lockout | Allowed after window |
| `separate limits per IP when no key` | Multiple IPs | Per-IP fallback | IP-spoofing bypass | Each IP gets own bucket |
| `limit shared across worker instances` | Two requests, simulated KV | Distributed limit | Per-instance counting = N×limit | **Integration test (per second-pass REWORK-9.1)** — real distributed limiting requires real KV. Move to `src/integration.test.ts` with Miniflare KV; unit-test only the local in-memory fallback path. Combined count enforces limit |

### `src/usage.test.ts` (new)

| Test | What | Value added | Failure mode protected | Expected |
|---|---|---|---|---|
| `logs token count from response` | Stub upstream returns usage | Billing depends on this | Missing log = free inference | KV/Durable Object write with token count |
| `logs even on partial failure` | Upstream 500 mid-stream | Failed requests still consume input tokens | Unbilled input tokens | Input tokens logged; output=0 |
| `does not double-log on retry` | Retry after 503 | Retry idempotency | Double billing | One log entry per logical request |
| `anonymises PII in log` | Request with API key in body | PII redaction in logs | Key leaked to log | Log entry has no raw key |
| `handles missing usage field` | Upstream returns no usage | Robustness | Worker crash on nil usage | Three rows (per second-pass REWORK-9.2): `usage` absent; `usage: null`; `usage: "not-an-object"`. Each logs 0 tokens; no crash |

### `src/circuit-breaker.test.ts` (new)

| Test | What | Value added | Failure mode protected | Expected |
|---|---|---|---|---|
| `opens after N consecutive failures` | N 500s | Upstream-down detection | 502 storm on upstream outage | Circuit opens; fast-fail |
| `half-open allows probe` | After open + cooldown | Recovery detection | Stuck open forever | One probe request allowed |
| `closes on probe success` | Probe 200 | Recovery | Stuck open after recovery | Circuit closes; traffic resumes |
| `stays open on probe failure` | Probe 500 | Hysteresis | Flap open/close | Re-opens; cooldown reset |
| `per-upstream isolation` | Two upstreams, one down | Multi-upstream | One bad upstream breaks all | Healthy upstream unaffected |

### `src/relay-mode.test.ts` (new)

| Test | What | Value added | Failure mode protected | Expected |
|---|---|---|---|---|
| `free-tier failure falls back to paid` | Free upstream 429 | Free → paid fallback | Users stuck on broken free tier | Paid upstream invoked |
| `paid-tier failure does not fall back` | Paid upstream 500 | No reverse fallback | Reverse fallback = cost surprise | Returns error; no free attempt |
| `mode recorded in log` | Any fallback | Audit trail | Silent fallback = unattributable cost | Log entry has `fallback=true` |

### `src/streaming.test.ts` (new)

| Test | What | Value added | Failure mode protected | Expected |
|---|---|---|---|---|
| `passes SSE chunks unchanged` | Stub sends 3 chunks | Streaming passthrough | Modified/dropped chunks | Client receives chunks in order |
| `closes stream on client disconnect` | Client kills connection | Resource cleanup | Leak on disconnect | Upstream connection closed |
| `handles upstream mid-stream error` | Upstream sends 200 then error | Mid-stream error handling | Silent truncation with 200 | Error surfaced to client |
| `does not buffer entire stream` | Large streamed response | Memory bound | OOM on large response | Memory bounded; chunks forwarded |

### `src/secret-rotation.test.ts` (new)

| Test | What | Value added | Failure mode protected | Expected |
|---|---|---|---|---|
| `old and new secret both valid during rotation` | Both secrets in env | Zero-downtime rotation | Active requests fail mid-rotation | Both accepted |
| `missing secret returns 403 not 500` | Empty env | Fail-closed | Open when misconfigured | 403 |

> **Note (second-pass audit DROP-9.1):** `old secret rejected after
> rotation complete` dropped — it mutates global worker state and races
> with parallel tests. The rotation flow is covered by the
> both-secrets-valid test plus the production rotation exercised by T3
> canaries in US-52.10 (which have their own isolated namespace).

### `src/body-limit.test.ts` (new)

| Test | What | Value added | Failure mode protected | Expected |
|---|---|---|---|---|
| `rejects body over limit` | POST with 11 MiB body | OOM prevention | Worker OOM on large body | 413 |
| `allows body at limit` | POST with exactly 10 MiB | Boundary correctness | Off-by-one rejection | 200 (or upstream's response) |
| `content-length missing rejected` | POST no Content-Length | Streaming upload safety | Streaming upload bypasses limit | 411 or 413 |

## Failure-Mode Coverage Matrix

| Failure mode | Covered by |
|---|---|
| Cost runaway (no rate limit) | ratelimit × 3 |
| Free inference (no usage log) | usage × 2 |
| 502 storm on upstream outage | circuit-breaker × 3 |
| Users stuck on broken free tier | relay-mode × 1 |
| Silent stream truncation | streaming × 2 |
| Worker OOM on large body / large stream | body-limit × 2, streaming × 1 |
| Rotation breaks active requests | secret-rotation × 1 |
| Open-when-misconfigured | secret-rotation × 1 |
| Per-instance counting under-counts | ratelimit × 1 |
