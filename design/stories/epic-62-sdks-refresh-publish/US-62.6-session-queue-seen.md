## User Story

As an SDK consumer, I want to enqueue messages when a session is busy, list and dismiss queued messages, and mark sessions as seen â€” so that I can build robust message-handling workflows without hand-crafting HTTP requests.

## Background

The session queue subsystem (Epic 41) is a first-class feature: when a session is busy (processing a turn), users can enqueue messages that drain automatically once the session goes idle. The queue endpoints have been live since Epic 41 but no SDK exposes them. The mark-seen endpoint (`PUT /sessions/{sessionId}/seen`) is a session-metadata operation used for unread-state tracking.
