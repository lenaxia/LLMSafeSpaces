# US-62.9: Contract Tests — `/message` Blocking + JSON Shape; Expand Hurl Suite

**Epic:** 62 - SDK Refresh, API-Surface Parity & Publish
**Priority:** Critical
**Depends on:** US-62.2 (Python SDK parity — the test harness uses the Python SDK's `send_message`)

## User Story

As an SDK maintainer, I need a contract test that proves `POST /sessions/{id}/message` blocks until the assistant turn completes and returns a single JSON object (not an SSE stream), so that the foundational assumption underlying every SDK's `send_message` implementation is validated against a real workspace before any SDK is published.

## Background

Every SDK's `send_message` method assumes EA3 (epic-level assumption): the call is synchronous and returns one JSON object. The API server evidence supports this (`proxy_test.go:2457` labels `SendMessage` "synchronous", asserts it does NOT get the 409 guard; returns 200 not 202). However, the residual risk is real: `doProxy` (`proxy.go:537`) streams 2xx responses chunk-by-chunk and has explicit SSE handling (`proxy.go:548`). If opencode ever returns `text/event-stream` for `POST /session/{id}/message`, every SDK's `resp.json()` / `json.Unmarshal` / `JSON.parse` / `Gson.fromJson` breaks silently.

This story is the epic's highest-risk retirement. It should be scheduled as the **first** story after US-62.2 lands — if EA3 is wrong, every SDK's `send_message` return type changes, which is a breaking change that blocks publish (US-62.8).

This story also expands the existing Hurl contract suite (US-14.7 created 3 files but they are not run in CI) to cover the new SDK surface added by US-62.6 and US-62.7.

## Acceptance Criteria

### Part 1: `/message` blocking + JSON contract test (Critical Path)

- [ ] A test exists that creates a real workspace via the Python SDK, ensures a session, sends a message, and asserts:
  - [ ] (a) The response is parseable as a single JSON object with a `parts` array — NOT an SSE stream. Proof: assert `Content-Type` response header does NOT start with `text/event-stream`, and the response body parses as JSON without error.
  - [ ] (b) The extracted text content (`MessageResponse.content`) is non-empty. This is the real proof the call blocked — if it returned immediately with empty content, the turn did not complete.
  - [ ] (c) Note: do NOT assert on wall-clock elapsed time. A fast model on a fast cluster could return a valid complete response in < 500ms; a timing threshold would produce flaky failures. Content-non-empty is the correct blocking proof.
- [ ] The test runs against a real workspace (kind cluster with a real opencode instance), not a mock. This is the entire point — mocks cannot settle whether opencode returns SSE.
- [ ] The test is tagged or placed so it runs in the nightly e2e suite (`local/test.sh` or a Go e2e test in `tests/`), not in the unit-test gate (it requires a cluster + real LLM credentials).
- [ ] If the test fails (EA3 is wrong), the failure message explicitly states: "EA3 invalidated: POST /message returns SSE, not JSON. All SDK send_message implementations must be redesigned." — so the failure is actionable, not opaque.

### Part 2: Expand Hurl contract suite

- [ ] Hurl files in `sdks/tests/contract/` cover the session queue surface (enqueue, list, dismiss) — 3 new cases minimum
- [ ] Hurl files cover error responses for queue operations (enqueue on non-existent workspace, dismiss non-existent message) — 2 new cases
- [ ] Hurl files cover the mark-seen endpoint — 1 new case
- [ ] Hurl files cover the session delete endpoint — 1 new case (happy path) + 1 case (delete on non-existent session)
- [ ] The `make contract-test-mock` target (Makefile:42) runs successfully with all new Hurl files against the Prism mock server built from the refreshed OpenAPI spec (US-62.1)
- [ ] Total contract test cases ≥ 30 (the US-14.7 acceptance criterion, now actually met)

### Part 3: Wire contract tests into CI

- [ ] `make contract-test-mock` runs in CI (`.github/workflows/`) on every PR that touches `sdks/` or `openapi.yaml`
- [ ] The `/message` live contract test runs in the nightly e2e (`.github/workflows/e2e-nightly.yml` or equivalent)

## Technical Details

### Part 1: Live `/message` contract test

**Location:** `tests/sdk_contract/message_blocking_test.go` (Go e2e test using the kind cluster, consistent with `tests/epic26/` and `tests/gharouter/` placement). Alternatively, a Python script invoked by `local/test.sh` if that is more natural for the e2e pipeline.

**Test shape (Python, if invoked by `local/test.sh`):**
```python
import httpx
from llmsafespaces import LLMSafeSpaces

def test_message_blocks_and_returns_json():
    client = LLMSafeSpaces(BASE_URL, api_key=API_KEY)
    ws = client.workspaces.create(name="ea3-contract", runtime="base", storage_size="1Gi")
    try:
        canary_wait_active(client, ws.id)  # poll until Active
        sess = client.sessions.ensure(ws.id)

        # SDK-level: content must be non-empty (proves blocking)
        result = client.sessions.send_message(ws.id, sess.session_id, "Reply with exactly: EA3-OK")
        assert result.content, "content is empty — send_message did not block for the response"

        # Raw HTTP level: Content-Type must NOT be text/event-stream (proves JSON, not SSE)
        resp = httpx.post(
            f"{BASE_URL}/api/v1/workspaces/{ws.id}/sessions/{sess.session_id}/message",
            headers={"Authorization": f"Bearer {API_KEY}"},
            json={"content": "Reply with exactly: EA3-OK",
                  "parts": [{"type": "text", "text": "Reply with exactly: EA3-OK"}]},
            timeout=120,
        )
        content_type = resp.headers.get("content-type", "")
        assert not content_type.startswith("text/event-stream"), \
            f"EA3 INVALIDATED: /message returns SSE, not JSON. " \
            f"All SDK send_message implementations must be redesigned. " \
            f"Content-Type: {content_type}"
        body = resp.json()  # must not raise
        assert "parts" in body, "response JSON missing 'parts' array"
    finally:
        client.workspaces.delete(ws.id)
```
