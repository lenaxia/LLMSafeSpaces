# US-62.2: Python SDK Typed-Surface Parity

**Epic:** 62 - SDK Refresh, API-Surface Parity & Publish
**Priority:** Critical
**Depends on:** US-62.1 (OpenAPI spec refresh — the spec is the contract this story implements against)

## User Story

As a Python developer integrating with LLMSafeSpaces (the external project), I want the Python SDK to expose every API surface the platform offers with typed methods, so that I can use the full platform capability without hand-crafting HTTP requests or forking the SDK.

## Background

The Python SDK at `sdks/python/llmsafespaces/` is the SDK the external integrator chose. It has drifted significantly from the API server. The sync client (`client.py`) already has `_AccountAPI` (`client.py:294-313`), `_PromptsAPI` (`client.py:389-414`), and `_AgentRolesAPI` (`client.py:417-463`) — all wired in the constructor (`client.py:50-57`). What it is missing: `UserSettingsService`, `ProviderCredentialsService`, `AdminProviderCredentialsService`, `sessions.delete()`, and the `__version__` attribute. The async client (`async_client.py`) is further behind (covered by US-62.3).

The Go SDK (`sdks/go/services.go`) is the style reference — it has 11 service groups. This story brings Python to parity with the router surface, matching Go's structure where it exists and filling Go's own gaps (provider-credentials) where Go is incomplete.

## Technical Details

### New service classes to add to `client.py`

**`_UserSettingsAPI`** — mirrors Go's `UserSettingsService` (`services.go:292-310`):
```python
class _UserSettingsAPI:
    def get(self) -> dict[str, Any]: ...
    def get_schema(self) -> dict[str, Any]: ...
    def set(self, key: str, value: Any) -> dict[str, Any]: ...
```
Routes: `GET /users/me/settings`, `GET /users/me/settings/schema`, `PUT /users/me/settings/{key}`

**`_AccountAPI`** — already exists at `client.py:294-313`. Verify against the refreshed spec. No changes expected unless the spec revealed a discrepancy.

**`_ProviderCredentialsAPI`** — new. 8 methods matching `router.go:429-439` (no Update — users delete+recreate). Methods returning single credentials use the `ProviderCredential` dataclass (matching Go's typed return); list methods return `list[ProviderCredential]`; void operations return `None`:
```python
class _ProviderCredentialsAPI:
    def create(self, *, name, kind, slug, api_key, base_url="") -> ProviderCredential: ...
    def list(self) -> list[ProviderCredential]: ...
    def get(self, cred_id: str) -> ProviderCredential: ...
    def delete(self, cred_id: str) -> None: ...
    def probe_models(self, cred_id: str) -> dict[str, Any]: ...  # opencode-proxied shape
    def list_bindings(self, cred_id: str) -> dict[str, Any]: ...
    def bind(self, cred_id: str, workspace_id: str) -> dict[str, Any]: ...
    def unbind(self, cred_id: str, workspace_id: str) -> None: ...
```
Note: `probe_models`, `list_bindings`, and `bind` return `dict` because their response shapes are not stable enough to type (opencode-proxied or simple key-value). `create`, `get`, and `list` return typed `ProviderCredential` for parity with Go's `ProviderCredentialResponse`.

**`_AdminProviderCredentialsAPI`** — new. 9 methods matching `router.go:405-417`.

### `__version__` in `__init__.py`

```python
import importlib.metadata

try:
    __version__ = importlib.metadata.version("llmsafespaces")
except importlib.metadata.PackageNotFoundError:
    __version__ = "dev"
```

Add `__version__` to `__all__`.

### Types to add to `types.py`

- `ProviderCredential` dataclass matching the Go `ProviderCredentialResponse` (`types.go:352-363`): `id`, `name`, `kind`, `slug`, `base_url`, `model_allowlist`, `model_context_limits`, `model_output_limits`, `created_at`, `updated_at`
- `UserSettings` dataclass: `settings: dict`, `schema_version: int`

### Test additions (`tests/test_client.py`)

Follow the existing `@respx.mock` pattern. Minimum one test per new method:
```python
@respx.mock
def test_provider_credentials_create():
    respx.post(f"{BASE}/provider-credentials").respond(
        status_code=201,
        json={"id": "cred-1", "name": "my-key", "kind": "openai", "slug": "my-key", ...},
    )
    client = LLMSafeSpaces("http://localhost:8080", api_key="lsp_test")
    result = client.provider_credentials.create(
        name="my-key", kind="openai", slug="my-key", api_key="sk-..."
    )
    assert result["id"] == "cred-1"
```

Also add `test_version()`:
```python
def test_version():
    import llmsafespaces
    assert llmsafespaces.__version__ != "unknown"  # at minimum, must be importable
```

### What NOT to change

- The `_request` / `_request_with_retry` infrastructure — it is correct and reusable
- The error hierarchy (`errors.py`) — unchanged
- The async client — US-62.3 handles that separately
- The `types.py` existing dataclasses — only add new ones
