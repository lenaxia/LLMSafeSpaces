# US-62.4: TypeScript SDK Typed-Surface Parity

**Epic:** 62 - SDK Refresh, API-Surface Parity & Publish
**Priority:** High
**Depends on:** US-62.1 (OpenAPI spec refresh — the contract to implement against)

## User Story

As a TypeScript/Node.js developer building tools on LLMSafeSpaces, I want the TypeScript SDK to expose every API surface the platform offers with typed methods, so that I can use the full platform capability with compile-time type safety.

## Background

The TypeScript SDK (`sdks/typescript/src/client.ts`) is close to the Go SDK in coverage but has specific gaps: missing `sessions.delete()`, missing the entire `ProviderCredentialsService` and `AdminProviderCredentialsService`, and missing `agentRoles.clearWorkspaceRole()` (the DELETE on workspace role). It already has `UserSettingsAPI`, `AccountAPI`, `PromptsAPI`, and `AgentRolesAPI`.

## Acceptance Criteria

- [ ] `SessionsAPI.delete(workspaceId, sessionId)` exists, calling `DELETE /workspaces/${workspaceId}/sessions/${sessionId}`
- [ ] `AgentRolesAPI.clearWorkspaceRole(workspaceId)` exists, calling `DELETE /workspaces/${workspaceId}/agent-role`
- [ ] `ProviderCredentialsAPI` class exists with: `create(req)`, `list()`, `get(id)`, `delete(id)`, `probeModels(id)`, `listBindings(id)`, `bind(credId, workspaceId)`, `unbind(credId, workspaceId)` (8 methods — no Update)
- [ ] `AdminProviderCredentialsAPI` class exists with: `list()`, `create(req)`, `get(id)`, `update(id, req)`, `delete(id)`, `probeModels(id)`, `createAutoApply(id, req)`, `listAutoApply(id)`, `deleteAutoApply(id, targetType, targetId)` (9 methods)
- [ ] Types are added to `src/types.ts` for `ProviderCredential`, `CreateProviderCredentialRequest`, `UpdateProviderCredentialRequest` (matching Go's `ProviderCredentialResponse` at `types.go:352-363`)
- [ ] Both new services are wired in the `LLMSafeSpaces` constructor (`client.ts:41-49`)
- [ ] Every new method has at least one unit test in `tests/client.test.ts` using the existing `vi.fn()` mock-fetch pattern
- [ ] `npm run typecheck` passes
- [ ] `npm run test` passes — no regressions

## Assumptions
