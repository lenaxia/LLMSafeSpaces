# US-62.5: Java SDK Typed-Facade Rewrite

**Epic:** 62 - SDK Refresh, API-Surface Parity & Publish
**Priority:** Normal
**Depends on:** US-62.1 (OpenAPI spec refresh вЂ” the contract to implement against)

## User Story

As a Java/JVM developer building integrations with LLMSafeSpaces, I want a typed Java SDK with domain methods, typed models, and typed exceptions, so that I can use the platform with compile-time safety instead of hand-crafting HTTP requests via a generic wrapper.

## Background

The current Java SDK (`sdks/java/src/main/java/com/llmsafespaces/sdk/LLMSafeSpacesClient.java`) is 133 lines of generic HTTP wrapper. It has `get(path, type)`, `post(path, body, type)`, `delete(path)` вЂ” no domain methods, no typed exceptions (all errors collapse to `LLMSafeSpacesException`), no auth-retry, no 404/409/429 differentiation. It uses Gson for serialization and Java 17's `java.net.http.HttpClient`.

This story rewrites it into a typed facade matching the other three SDKs' service-group structure. The alternative outcome (deprecation) is explicitly permitted by the epic risk table вЂ” if the team decides the maintenance cost of a fourth SDK is not justified, this story becomes "deprecate Java, update `sdks/README.md`, remove the Makefile target."

## Acceptance Criteria

- [ ] **Decision recorded:** either (a) the typed-facade rewrite is implemented (criteria below) or (b) the Java SDK is deprecated with `sdks/README.md` updated, Makefile target removed, and the decision documented in a worklog
- [ ] If rewrite: `LLMSafeSpacesClient` is a builder-constructed facade exposing typed service groups: `workspaces()`, `sessions()`, `auth()`, `secrets()`, `terminal()`, `account()`, `userSettings()`, `providerCredentials()`, `adminProviderCredentials()`, `prompts()`, `agentRoles()`
- [ ] If rewrite: typed exception hierarchy: `LLMSafeSpacesException` (base) в†’ `AuthException` (401/403), `NotFoundException` (404), `ConflictException` (409), `RateLimitException` (429) вЂ” matching the other SDKs' error maps
- [ ] If rewrite: typed model classes for `Workspace`, `WorkspaceListResult`, `EnsureSessionResponse`, `MessageResponse`, `SecretResponse`, `APIKey`, `TerminalTicket`, `ProviderCredential`
- [ ] If rewrite: `sessions.sendMessage(workspaceId, sessionId, content)` returns `MessageResponse` with `getContent()` (extracted text) and `getRaw()` (parsed JsonObject)
- [ ] If rewrite: auth supports API key (Bearer header), token (from login), and auto-login with email+password
- [ ] If rewrite: 401 auto-retry with re-login (matching the sync Python and TypeScript clients)
- [ ] If rewrite: every service method has at least one JUnit 5 test using a mock HTTP server (`com.sun.net.httpserver.HttpServer` or WireMock)
- [ ] If rewrite: `pom.xml` gains no new runtime dependencies beyond Gson (already present)

### Exception hierarchy

Exceptions extend `RuntimeException` (unchecked), not `Exception` (checked). Checked exceptions force every caller to declare `throws` or wrap in try/catch, which is non-idiomatic for HTTP client SDKs вЂ” the JDK's own `HttpClient` uses `IOException` (checked) but modern Java SDKs (Spring WebClient, Retrofit, gRPC stubs) have moved to unchecked exceptions for ergonomics. The other three SDKs use unchecked exceptions (Python exceptions are always unchecked; Go uses error values; TypeScript uses Error).

```java
public class LLMSafeSpacesException extends RuntimeException {
    private final int statusCode;
    // ...
}

public class AuthException extends LLMSafeSpacesException { ... }
public class NotFoundException extends LLMSafeSpacesException { ... }
public class ConflictException extends LLMSafeSpacesException { ... }
public class RateLimitException extends LLMSafeSpacesException { ... }
```

The `request()` method maps status codes:
```java
switch (statusCode) {
    case 401, 403 -> throw new AuthException(msg, statusCode);
    case 404 -> throw new NotFoundException(msg);
    case 409 -> throw new ConflictException(msg);
    case 429 -> throw new RateLimitException(msg);
    default -> throw new LLMSafeSpacesException(msg, statusCode);
}
```

### Test pattern

```java
@Test
void workspacesCreate_returnsTypedWorkspace() throws Exception {
    HttpServer server = HttpServer.create(new InetSocketAddress(0), 0);
    server.createContext("/api/v1/workspaces", exchange -> {
        String body = """
            {"id":"ws-1","name":"test","userId":"u1","runtime":"base",
             "storageSize":"10Gi","phase":"Pending",
             "createdAt":"2026-01-01T00:00:00Z","updatedAt":"2026-01-01T00:00:00Z"}""";
        exchange.sendResponseHeaders(201, body.length());
        exchange.getResponseBody().write(body.getBytes());
        exchange.close();
    });
    server.start();

    var client = LLMSafeSpacesClient.builder("http://localhost:" + server.getAddress().getPort())
        .apiKey("lsp_test").build();
    var ws = client.workspaces().create(new CreateWorkspaceRequest("test", "base", "10Gi"));
    assertEquals("ws-1", ws.getId());

    server.stop(0);
}
```
