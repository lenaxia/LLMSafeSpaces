# US-62.3: Python SDK — Async Client Parity with Sync

**Epic:** 62 - SDK Refresh, API-Surface Parity & Publish
**Priority:** High
**Depends on:** US-62.2 (Python sync parity — the new sync services must exist before mirroring to async)

## User Story

As a Python developer using async-native frameworks (FastAPI, LangChain async, asyncio pipelines), I want the async SDK client to expose every service the sync client does, so that I can use the full platform capability without blocking the event loop.

## Background

The async client (`async_client.py`) is currently at parity with the sync client — both expose 8 service groups (workspaces, sessions, auth, account, secrets, terminal, prompts, agent_roles). Once US-62.2 adds `UserSettingsService`, `ProviderCredentialsService`, `AdminProviderCredentialsService`, and `sessions.delete()` to the sync client, the async client falls behind. This story mirrors those additions.

## Acceptance Criteria

- [ ] `AsyncLLMSafeSpaces` exposes `user_settings`, `provider_credentials`, `admin_provider_credentials` service groups matching the sync client's signatures
- [ ] `async_sessions.delete(workspace_id, session_id)` exists
- [ ] Every async method is the `await self._c._request(...)` analog of the sync `self._c._request(...)` call — same paths, same bodies, same return types
- [ ] Every new async method has at least one unit test using `respx.mock` + `asyncio` (matching the existing pattern in `tests/test_async_client.py`)
- [ ] Existing async tests continue to pass — no regressions
- [ ] The async `_request` / `_request_with_retry` infrastructure is reused unchanged — no new auth, retry, or error-mapping logic

## Technical Details

### What to add

Mirror each new sync service class as an `_Async<Service>` class:

1. **`_AsyncUserSettingsAPI`** — async version of `_UserSettingsAPI`: `get()`, `get_schema()`, `set(key, value)`
2. **`_AsyncProviderCredentialsAPI`** — 8 methods (no Update — see US-62.2 A4)
3. **`_AsyncAdminProviderCredentialsAPI`** — 9 methods (includes Update)
4. **`_AsyncSessionsAPI.delete()`** — add `await self._c._request("DELETE", ...)`

Wire each in the constructor (`async_client.py:55-62`), matching the sync client's wiring order.

### Test pattern

```python
@respx.mock
async def test_async_provider_credentials_create():
    respx.post(f"{BASE}/provider-credentials").respond(
        status_code=201,
        json={"id": "cred-1", ...},
    )
    client = AsyncLLMSafeSpaces("http://localhost:8080", api_key="lsp_test")
    result = await client.provider_credentials.create(...)
    assert result["id"] == "cred-1"
```
